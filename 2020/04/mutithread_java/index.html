多进程只能串行执行（宏观上看）

  • 每个时间片内将cpu分配给某一个任务，时间片结束，cpu将自动回收，在分配给另外任务，从外部看，所有任务都是同时执行，但是在cpu上，任务是按照串行依次运行（单核cpu）。如果是多核，多个任务可以并行，但是单个核上，多进程只能串行执行。
  • 优点：可以同时运行多个任务，程序因IO堵塞时，可以释放cpu，让cpu执行其他服务，当系统有多个cpu时，可以为多个程序同时服务
  • 缺点：太笨重，不好管理，不好切换

  • 多线程（并行程序）

    • 一个程序可以包括多个子任务，可串/并行

    • 每个只任务可以成为一个线程 -如果一个子任务阻塞，程序可以将cpu调度另外一个子任务进行工作。这样cpu还是保留在本程序中，而不是被调度到别的程序去，这样提到本程序所获得cpu时间和利用率

      多进程和多线程的区别

  • 线程共享数据

  • 线程通讯更高效

  • 线程更轻量级，更容易切换

  • 多个线程更容易管理

    多线程的创建

    java.lang.Threadl类

java.lang.Runnable接口

  • 多线程的启动

  • stat方法，会自动以新进程调用run方法

  • 直接调用run方法，将变成串行执行

  • 同一个线程，多处stat会报错，只执行第一次stat方法

  • 多个线程启动，其启动的先后顺序是随机的

  • 线程无需关闭，只要其run方法执行结束后，自动关闭

  • main函数（线程）可能早于新线程结束，整个程序并不终止

  • 整个程序终止是所有的线程都终止（包活main函数线程）

  • Thread vs Runnable

    • Thread 占据类父类的名额，不如Runnable方便,Java是单根继承，一旦继承了一个类，无法继承及其类
    • Thread 类实现了Runnable
    • Runnable启动时需要Thread类的支持

    • Runnable更容易实现多线程中的资源共享

      多线程信息共享

  • 粗粒度：子线程与子线程之间，和mmain之间缺乏交流

  • 细粒度：线程之间有信息交流通讯

  • 多线程信息共享问题

    • 工作缓存副本（锁，同时读写一个数据）：采用volatile关键字修饰变量，保证不同线程对共享变量操作时的可见性
    • 关键步骤缺乏枷锁限制

  • 互斥：某一个线程运行一个代码段（关键去）， 其他线程不能同时运行这个代码段 2. 同步：多个线程的运行，必须按照某一种规定的先后顺序来运行，互斥是同步的一种特例

  • 关键字：synchronized，加大性能负担，但是使用便捷

锁

  • 多线程死锁
    • 每一个线程互相持有别人需要的锁，预防死锁，对资源进行等级排序(统一顺序锁获得r1,r2 )

  • 守护（后台）线程

    • 守护线程永远不要访问资源，如文件或数据库等
    • 普通线程的结束，是run方法运行结束

    • 守护线程的结束，是run方法运行结束，或main函数

多线程管理

  • 线程被动多暂停和终止

    • 没有及时释放资源，依靠别的线程来处理

      class TestThread1 extends Thread {
TestThread1() {
}

public void run() {
while(true) {
    if (!interrupted()) {
        System.out.println("test thread1 is running");
        try {
            Thread.sleep(1000);//需自己处理异常，还可能资源无法释放
            continue;
        } catch (InterruptedException var2) {
            var2.printStackTrace();
        }
    }

    System.out.println("test thread1 is exiting");
    return;
}
}
}

  • 线程主动暂停和终止

    • 定期检测共享变量
    • 如果需要暂停或终止，先释放资源，在主动动作
    • 暂停：Thread.sleep() 休眠

    • 终止：run方法结束，线程终止

      class TestThread2 extends Thread {
public volatile boolean flag = true;

TestThread2() {
}

public void run() {
while(this.flag) {
    System.out.println("test thread2 is running");

    try {
        Thread.sleep(1000L);
    } catch (InterruptedException var2) {
        var2.printStackTrace();
    }
}

System.out.println("test thread2 is exiting");
}
}
public class InterruptTest {
public InterruptTest() {
}

public static void main(String[] args) throws InterruptedException {
TestThread1 t1 = new TestThread1();
TestThread2 t2 = new TestThread2();
t1.start();
t2.start();
Thread.sleep(2000L);
t1.interrupt();
t2.flag = false;
System.out.println("main thread is exiting");
}
}
