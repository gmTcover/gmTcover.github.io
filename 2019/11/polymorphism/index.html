Polymorphism

十一月 5, 2019 发布在 c++基础, Class

polymorphism

  • Inheritance in C++

  • Can define one class in terms of another

    class XYPos {...};//x,y point

class Shape {
public:
	Shape();
	virtual ~Shape();//virtual 虚函数
	virtual void render();//意思是将来shape类的所以子类如果重新写了rander，这两个函数就有联系
	void move(const XYPos&E);
	virtual void resize();

protected:
	XYPos center;
};


class Ellipse : public Shape {
public:
	Ellipse( float maj, float minr );
	virtual void render();// will define own,继承过来的 同名肯定是virtual，这边写上是习惯
protected:
	float major_axis, minor_axis;

}

class Circle : public Ellipse {
public:
	Circle( float radius ) : Ellipse( radius, radius ) { }
	virtual void render();

}

void render( Shape* p) {
	p->render();//p指向虚函数，P是多态的，逻辑上子类地址或引用，底层对应父类数据结构，实现运行时刻调用各自的render
}//call correct render function ,p的类型是静态的（shape），p所指类型是动态的（原理rander（）是virtual）

void func() {
	Ellipse ell(10 ,20 );
	ell.render();
	Circle circ(40);
	circ.render();
	render(&ell);//upcast，参数，把子类指针赋值给父类，，运行时刻才能确定类型
	render(&circ);//upcast 可通过引用和指针
}
C++两种多态性
编译时多态性：通过重载函数实现
运行时多态性：通过虚函数实现

多态建立的基础

  • Upcast：take an object of the derived class as an object of the base one

  • Dynamic binding:

    • Binding: whic function to be called
    • Static binding: call the function as the code

    • Dynamic binding: call the function of the object

      C++ 都能c实现

  • 一定没有黑魔法，不会实现的很复杂，效率会很低

    virtual destructors

  • Make destructors virtual if they might be inherited

  • 如果类里面有一个virtual函数，它的析构函数必须是析构函数，将来程序修改有子类

  • 所有oop语言默认是动态绑定（追求语言），只有c++默认是静态绑定（提高效率，编译时刻）

  • C++中基类采用virtual destructor是为了防止内存泄漏

  • 只有一个类被用来当作基类的时候，才会把析构函数写成虚函数

    override

  • 覆盖，改写

    class Base {
public:
	virtual void func();

};
class Derived : public Base {
public:
	virtual void func();
	//override Base::func()

};

  • You can still call the overridden function

    void Derived::func(){
	cout << "In Dervied::func!";
	Base::func();//call to base class
}

  • This is a commen way to add new functionality

  • No need to copy the old stuff

    Return types relaxtion(current)

  • Suppose D is publicly derived from B

  • D::f() can return a subclass of the return type defined in B::f()

    • 子类的返回类型只有是父类数据类型，内存模型时

    • 才得到父类的vtable数据结构，实现运行时多态
