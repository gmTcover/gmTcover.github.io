拷贝构造

十一月 9, 2019 发布在 c++基础, Class

copying

  • creat a new object from an existing one

    • For example, when calling a function

      //Currency as pass-by-value argument

void func( Currency p) {//参数为对象Currency
	cout << "X = " << p.dollars();

}

...

Currency bucks( 100, 0 );

Currency p = bucks;// 初始化，前面有变量名，定义变量名的时候给了一个值，初始化

p = bucks;//赋值，编译器会自动调用函数，做赋值运算

C++ 中必须分清

  • declaration，definition

  • initialization ， assignment （c中可以当作没有区别，c++完全不同的两件事）

    拷贝构造特征

  • Copying is implemented by the copy constructor

  • Has the unique signature

    T::T(const T&)//传参数过程不会发生拷贝构造，没有制造新的对象出来 ，只是reference，安全的

T::T (T a);//传同类型参数过程编译器自动拷贝构造参数，a是实例化对象时，用拷贝构造函数初始化时，a无法得到初始化，为了完成第一步初始化，不断往内部调用拷贝构造函数，递归，error，

  • call-by-reference is used for the explicit argument

  • 自动提供（编译器）

  • C++ builds a copy constructor for you if you don’t provude one!

    • Copies each menber,object,arrays（ Good for numbers,objects,arrays）
    • Copies each pointer( Data may become shared )

  • 假如成员有指针的话，发生指针的拷贝

    • 获得2该指针指向同一块内存（直接不写拷贝构造会多次析构，error），2高级指针,指针copy在调用析构函数（需要malloc，new一块新的空间，不让他指向同一块区域），指针bit级别

    • 推荐字符转是用c++专门的sting,string是c++中标准库的类，一定有拷贝构造函数

      #include<string>
class Person {
	public:
	Person( const string& );
	~Person();
	void print();
	private:
	string name; //embedded object
	//...other data members
}

  • 你提供了拷贝构造函数，可以控制如何进行拷贝（选择性拷贝）

  • Copies and overhead

  • Compliers can “optimize out ” copies when safe!

    Person copy_func( char *who ) {
	Person local( who );
	local.print();
	return local;// copy ctor called
}

Person nocopy_func( char *who ) {
	return Person ( who );//no copy needed! 编译器自动优化，local对象没有使用，相当于只用who构造了 Person;这里不是拷贝构造，一步到位，编译器优化
}

Construction Vs. assignment

  • Every object is constructor once
  • Every object should be destroy once
    • Failur to invoke delete()
    • Invoke delete() more than once
  • Once an object is constructed, it can be the target of many assignment operations
